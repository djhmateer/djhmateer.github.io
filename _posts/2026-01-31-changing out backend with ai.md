## JavaScript

However I know that it is brittle as JavaScript - not a great backend language choice

## TypeScript

TypeScript solves the type safety problem but not the ecosystem problem. 

This feels like an incremental fix.

## C#

C# is a safe, competent choice. You know it, it's stable, types are good. 

The tradeoff: you'll spend energy resisting enterprise patterns and steering Claude away from OO. If you're okay with that discipline, it works. If you want the language/ecosystem to naturally support simplicity, Go is a better fit
