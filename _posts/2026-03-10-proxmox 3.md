- /dev/nvme0n1 – 2 TB (KINGSTON)
- /dev/nvme1n1 – 500 GB (ADATA LEGEND) - boot drive


LVM - Logical Volume Manager

Instead of

Physical disk -> partition -> filesystem

We have

Physical Disk → Physical Volume (PV) → Volume Group (VG) → Logical Volumes (LVs)
Physical Disk (or partition) → PV → VG → LVs

The ADATA has 3 partitions has these for EFI and BIOS boot loaders
The Kingston won't need these partitions.

## Increase Disk Size

```bash
# allocate more space on hypervisor

# make the partition see the space (dev/sda3, Resize, Write, quit)
sudo cfdisk

# extend the physical volume from the partition
sudo pvresize /dev/sda3

# extend LV to use up all space from VG
sudo lvextend -l +100%FREE /dev/ubuntu-vg/ubuntu-lv

# resize file system
sudo resize2fs /dev/mapper/ubuntu--vg-ubuntu--lv

# check can see the space on filesystem
df -h
```
